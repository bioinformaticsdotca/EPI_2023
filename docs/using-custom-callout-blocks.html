Using Custom Callout Blocks

This guide explains how to use the custom callout blocks in your R Markdown documents. These callouts allow you to highlight information in various styles and colors, and can be either static or collapsible (dropdowns).

Assuming the necessary callouts_conditional.lua filter, style.css, and fa-header.html (for icons) are correctly set up in your Bookdown project’s YAML configuration and project directory (as specified in this document’s YAML header), you can create callouts using a simple Markdown syntax.

Basic Syntax Callouts are created using Pandoc’s fenced div syntax. The basic structure is:

:::: {.callout type="your_type" title="Your Title Here"}

Your content goes here. This can be **any** valid Markdown,
including paragraphs, lists, code blocks, etc.

::::

Important: Ensure there is a blank line after the opening :::: {….} line and a blank line before the closing :::: line, especially if the content within the callout includes block-level elements like code chunks or multiple paragraphs.

Available Attributes You can customize the appearance and behavior of the callout using attributes within the curly braces {}:

type=“COLOR” (Required)

Determines the primary color scheme of the callout.

Available values: “red”, “blue”, “green”, “yellow”, “purple”, “orange”, “gray”.

Example: type=“blue”

style=“STYLE_NAME” (Optional)

Specifies the visual style of the callout.

Available values:

“regular” (This is the default if the style attribute is omitted)

“important”

“subtle”

Example: style=“important”

title=“YOUR TITLE TEXT” (Optional)

Sets the title that appears at the top of the callout.

If omitted, dropdown callouts will default to “Details” as the title. Static callouts will simply not have a visible title bar if this is omitted.

Markdown within the title (like emphasis or bold) will be rendered.

Example: title=“Important Note!”

icon=“ICON_OPTION” (Optional)

Controls the icon displayed next to the title.

Available values:

“true” (Default if icon attribute is present without a specific value, or if attribute is omitted): Shows a default icon based on the type.

“false”: No icon will be displayed.

“fas fa-your-icon-name”: Specify any Font Awesome (v6 free set) solid icon class (e.g., “fas fa-rocket”, “fas fa-book-open”).

Example: icon=“fas fa-lightbulb”

center_title=“true” (Optional, for Static Callouts only)

If set to “true”, the title of a static (non-collapsible) callout will be centered.

This attribute has no effect on dropdown callouts, as their titles are part of the summary line.

Example: center_title=“true”

collapsible=“true” (Optional)

If set to “true”, the callout will be rendered as a collapsible dropdown (using HTML
and

elements). The title and icon (if any) will be in the clickable summary bar.

If omitted or set to any other value (e.g., “false”), the callout will be a static, always-visible block.

Example: collapsible=“true”

Examples of R Markdown Syntax Here are examples of how you would write these callouts in your .Rmd file.

Regular Callouts (Static) Blue Information Box with Default Icon:

Information

This is a standard blue informational callout. The default icon for ‘blue’ will be used.

Green Success Box with Custom Icon and Centered Title:

Task Complete!

Congratulations! The process finished successfully.

All data saved.

Report generated.

Important Callouts (Static) Red Important Warning:

CRITICAL WARNING

System integrity is compromised. Immediate action required. Please check the error_log.txt file for details.

Subtle Callouts (Static) Yellow Subtle Note without Icon:

Quick Reminder

Don’t forget to save your work frequently. This is a subtle yellow reminder.

Dropdown (Collapsible) Callouts Set collapsible=“true” to make any callout a dropdown.

Optional Advanced Details
